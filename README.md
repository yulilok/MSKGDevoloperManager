# Бот Менеджера Задач Разработчиков MSKG

Бот Discord для управления идеями задач разработчиков, одобрениями, назначениями и завершением с использованием Google Sheets для хранения.

## Настройка

1. **Создание бота Discord:**
   - Перейдите на [Портал Разработчиков Discord](https://discord.com/developers/applications).
   - Создайте новое приложение и бота.
   - Скопируйте токен бота и добавьте его в `.env` как `DISCORD_TOKEN=your_token_here`.
   - Пригласите бота на ваш сервер с соответствующими разрешениями (управление ролями, отправка сообщений и т.д.).

2. **Настройка Google Sheets:**
   - Создайте новую таблицу Google.
   - Настройте заголовки в таблице: ID, Idea, Status, Assignee, Priority, Deadline, SubmittedBy.
   - (Поделитесь ею с email сервисного аккаунта после создания аккаунта в шаге 3).

3. **API Google Sheets:**
   - Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
   - Включите Google Sheets API.
   - Создайте сервисный аккаунт и получите его email.
   - Поделитесь таблицей Google с этим email (из шага 2).
   - Скачайте JSON-ключ как `credentials.json` в корень проекта.
   - Скопируйте ID таблицы из URL и добавьте в `.env` как `SHEET_ID=your_sheet_id_here`.

4. **Настройка Ролей, Каналов и сервера для быстрого теста:**
   - В `config.py` замените плейсхолдеры ID ролей на реальные ID ролей Discord.
   - Установите ID каналов для идей и одобрений.
   - Добавьте `GUILD_ID` (ID вашего сервера) в `.env` для мгновенной регистрации слэш-команд при тесте.

5. **Запуск Бота:**
   - Установите зависимости: `pip install -r requirements.txt`
   - Запустите: `python bot.py`

## Команды

- `/submit_idea <idea>`: Отправить новую идею задачи (требуется роль отправителя).
- `/pending_tasks`: Просмотреть ожидающие одобрения задачи с кнопками (руководство).
- `/approve <task_id>`: Одобрить задачу (руководство).
- `/reject <task_id>`: Отклонить задачу (руководство).
- `/claim_task <task_id>`: Забрать одобренную задачу (разработчик).
- `/my_tasks`: Просмотреть свои задачи с кнопками завершения (разработчик).
- `/complete_task <task_id>`: Отметить задачу как завершенную (разработчик).
- `/accept_completion <task_id>`: Принять завершение (руководство).
- `/request_changes <task_id> <reason>`: Запросить изменения (руководство).
- `/set_priority <task_id> <priority>`: Установить приоритет (руководство).
- `/set_deadline <task_id> <deadline>`: Установить дедлайн (руководство).

## Функции

- Разрешения на основе ролей.
- Приоритеты и дедлайны задач.
- Уведомления через Discord.
