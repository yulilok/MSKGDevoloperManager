Discord-бот для управления задачами разработчиков: подача идей, одобрение, назначение и завершение задач с использованием Google Sheets для хранения данных.

## Описание

Бот предоставляет систему управления задачами для команд разработчиков с ролевой моделью:

- **Отправители** — могут предлагать новые идеи задач
- **Руководство** — рассматривают, одобряют/отклоняют задачи, назначают приоритеты и дедлайны
- **Разработчики** — берут задачи в работу, отмечают выполнение

Все задачи хранятся в Google Sheets с полной историей статусов и метаданными.

## Установка и настройка

### 1. Создание Discord-бота

1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications)
2. Создайте новое приложение
3. В разделе **Bot** создайте бота
4. Скопируйте токен бота
5. В разделе **OAuth2 → URL Generator** выберите разрешения:
   - `Send Messages`
   - `Embed Links`
   - `Use Application Commands`
   - `Manage Roles` (опционально)
6. Сгенерируйте ссылку и пригласите бота на сервер

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
DISCORD_TOKEN=your_bot_token_here
SHEET_ID=your_google_sheet_id_here
GUILD_ID=your_guild_id_here
CREDENTIALS_PATH=credentials.json
```

**Параметры:**
- `DISCORD_TOKEN` — токен вашего Discord-бота
- `SHEET_ID` — ID Google Sheets (из URL таблицы)
- `GUILD_ID` — ID сервера для быстрой регистрации слэш-команд (опционально)
- `CREDENTIALS_PATH` — путь к JSON-файлу учётных данных Google API

### 3. Настройка Google Sheets

1. Создайте новую Google Таблицу
2. В первой строке укажите заголовки столбцов:
   ```
   ID | Name | Description | Status | Priority | Deadline | Assignee | SubmittedBy | Role | RejectionReason
   ```
3. Настройте Google Sheets API:
   - Перейдите в [Google Cloud Console](https://console.cloud.google.com/)
   - Создайте проект и включите **Google Sheets API**
   - Создайте сервисный аккаунт (Service Account)
   - Скачайте JSON-ключ и сохраните как `credentials.json` в корне проекта
4. Откройте доступ к таблице для сервисного аккаунта:
   - В JSON-файле найдите поле `client_email`
   - В Google Sheets нажмите **Настройки доступа** и добавьте этот email с правами **Редактор**

### 4. Настройка каналов

В файле `config.py` укажите ID каналов:

```python
IDEAS_CHANNEL_ID = 123456789012345678   # Канал для подачи идей
APPROVAL_CHANNEL_ID = 123456789012345679  # Канал для рассмотрения руководством
TASKS_CHANNEL_ID = 123456789012345680     # Канал с доступными задачами
```

**Как получить ID канала:**
1. В Discord включите режим разработчика (Настройки → Дополнительно → Режим разработчика)
2. ПКМ по каналу → Копировать ID

### 5. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 6. Запуск бота

```bash
python bot.py
```

После запуска бот:
- Зарегистрирует слэш-команды на сервере (если указан `GUILD_ID`)
- Отправит сообщение с кнопкой в канал идей

## Команды

### Слэш-команды

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/my_tasks` | Просмотр своих активных задач | Разработчики |
| `/complete_task <task_id>` | Завершить задачу и отправить на проверку | Разработчики |
| `/accept_completion <task_id>` | Принять выполненную задачу | Руководство |
| `/request_changes <task_id> <reason>` | Запросить изменения в задаче | Руководство |

### Кнопочные команды

- **Отправить идею** — в канале идей (модальное окно)
- **Взять задачу** — в канале задач
- **Принять / Отклонить** — в канале одобрений
- **Завершить** — в списке своих задач

## Структура Google Sheets

Таблица должна содержать следующие столбцы:

| Столбец | Описание |
|---------|----------|
| `ID` | Уникальный идентификатор задачи (8 символов UUID) |
| `Name` | Название задачи |
| `Description` | Подробное описание |
| `Status` | Статус: `pending_approval`, `approved`, `in_progress`, `review`, `completed`, `rejected` |
| `Priority` | Приоритет: `Низкий`, `Средний`, `Высокий` |
| `Deadline` | Дедлайн в формате YYYY-MM-DD |
| `Assignee` | ID Discord-пользователя исполнителя |
| `SubmittedBy` | ID Discord-пользователя автора идеи |
| `Role` | Требуемая роль: `Скриптер`, `Билдер`, `Аниматор`, `Моделер` |
| `RejectionReason` | Причина отклонения (если отклонено) |

## Использование

### Типовой рабочий процесс

1. **Подача идеи:**
   - Участник нажимает кнопку «Отправить идею для разработки» в канале идей
   - Заполняет название и описание в модальном окне
   - Идея публикуется в канале одобрений

2. **Рассмотрение:**
   - Руководство видит идею в канале одобрений
   - Кнопки «Принять» или «Отклонить»
   - При принятии: выбирается роль, приоритет, дедлайн, заметки
   - Задача публикуется в канале задач

3. **Выполнение:**
   - Разработчик нажимает «Взять задачу»
   - Задача закрепляется за ним
   - По завершении: команда `/complete_task` или кнопка «Завершить»

4. **Проверка:**
   - Руководство получает уведомление в канале одобрений
   - Принимает задачу или запрашивает изменения
